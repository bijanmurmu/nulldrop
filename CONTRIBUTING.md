# Contributing to Nulldrop

Thank you for your interest in contributing! Nulldrop is built on principles of extreme minimalism, elegant design, and high performance. 

## Design Language Guidelines
If you are modifying the UI, please strictly adhere to the editorial design system:
1. **Colors:** Use only the 4 predefined editorial variables (`editorial-bg`, `editorial-fg`, `editorial-surface`, `editorial-accent`). No arbitrary HEX codes.
2. **Typography:** Rely on `Playfair Display` for bold, italicized headers and `Inter` for highly-tracked, uppercase technical data.
3. **Spacing:** Maintain massive amounts of negative space. Do not cram components.

## Pull Request Process
1. Fork the repository and create your feature branch (`git checkout -b feature/AmazingFeature`).
2. Ensure you have tested both the Frontend and Backend locally.
3. Commit your changes (`git commit -m 'Add some AmazingFeature'`).
4. Push to the branch (`git push origin feature/AmazingFeature`).
5. Open a Pull Request using the provided GitHub template.

## Issues and Bugs
Please use the GitHub Issue Tracker. We have provided templates for Bug Reports and Feature Requests to make this process seamless.
